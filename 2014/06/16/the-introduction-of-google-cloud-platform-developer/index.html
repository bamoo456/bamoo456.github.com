the introduction of google cloud platform developer

簡單記錄一下在google cloud platfrom介紹當天的一些筆記

去聽了開發人員的介紹以後，大概可以知道的就是他們現在很積極的推薦這個平台，

相對於amazon亦做了一些差異化的服務!

  • App Engine (PaaS)

  • Compute Engine (IaaS)

這兩個原有的服務在網路上就有很多相關的介紹了，所以這邊就只有記錄下面這項新的服務。

  • Managed VMs

在目前既有的app engine與compute engine架構下，為了增加google cloud platform的彈性!

通常，會使用app engine是為了讓developer能夠更專注在目前服務的開發，而管理與擴充的部分則交由Google去煩惱…

但遇到以下的情況時，就又會讓developer感到麻煩…

Example

簡單來講，如果目前正在run的服務是在app engine上，但又需要額外的c/c++ lib時(app engine不支援c/c++)

它可以透過Managed VMs來新增一台虛擬主機，然後讓c/c++ lib在這些Managed VMs上面執行!

而google cloud則會負責連接目前的app engine與Managed VMs上的Lib，並且作好擴充的準備!

不過實際上還是要試了才知道真正的使用效益大不大!

其它相關資訊：

  • google compute engine目前的’asia’指的就是台灣 !

  • google app engine服務中包含了 CDN, DNS… 相關

  • google endpoint 能夠很快速的產生client 與 google cloud API 連接的sample code
