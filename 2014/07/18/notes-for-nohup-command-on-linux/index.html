notes for nohup command on Linux

nohup

記錄一下這個linux的舊指令用法…

當我們正常執行指令是透過telnet或ssh時，通常會遇到遠端離線時，
同時正在執行的程式也跟著停掉了；

在某些情況下，這倒是蠻困擾人的，也許我們就只是想要遠端進去server讓程式執行整晚!
但卻因為程式無法繼續，而產生問題…

why command/program will be stopped when logout ?

主要在於：
當我們從一個連線登出的時候，每個process都會收到一個SIGHUP信號，透過nohup所下達的命令就可以不受影響，繼續執行。

=>詳細解釋出處

How to use nohup

最簡單的方式是在原本要執行的執令前面加入nohup
ex:

nohup your_command &


nohup your_command &>myout.txt &

這邊將原始指令的輸出重新導到myout.txt

Some tips

這個指令比較需要注意的是，如果我們原本要使用的指令會一直產生資訊在terminal上，
則使用了nohup後，它會將這些原始的輸出導到一個檔案上:nohup.out

這就會產生一個問題，如果長期使用nohup時，這個檔案就會一直增長下去
所以使用時，還需要注意這個問題 !
